------------------Async vs await------------------
1. async: used for telling the JS that this function will work with asynchronous operations and will always return a Promise.

When you put async before a function:
async function getFact() {
}

Example:

async function hello() {
    return "Hello";
}

it will prints "Hello"




2. Await: pause this async function until the promise finishes

Example:
let response = await fetch(url);
JavaScript waits for the API request to complete before moving to the next line.


Without await:
let response = fetch(url);
console.log(response);

You'll see a Promise object instead of the actual result.


*await* can only be used::
Inside an async function, or
At top level in environments that support top-level await.

So::
async function getFact() {
    let response = await fetch(url);
}

is valid.

---------------- Import module ----------------
before ES6 update, we used the method 'require', where we have to do the export the same way and then require the whole file, so, everything will load up, according to whatever we exported.
so, they introduce the new feature, 
The import module, it is the new technology and it gives few features like we do not have to import the whole file, we can just import the things of whatever is required for us, like if the file have thousands of line of code and we just need two to three function from it, so, we can just import those two to three function specifically but in that require case, we can't do that, we have to specifically load all of those functions in our project and it was not very efficient for memory.

loading is synchronous for 'require' but can be asynchronous for 'import'

but the most important part is, we can export and import the things from that normally, but we have to write 
{
"type": "module"
} 
in our package.json file to use it. 
and we do not have package.json file in our project, we can create it by the command with "npm init"
and then we have to enter that module thing in our project to run this properly.
