A normal HTML file is static:

But suppose you want:
User A → "Hello Shaan"
User B → "Hello Rahul"

You would need to generate HTML dynamically.

Express:
res.render("home.ejs", {

});

Output:
<h1>Hello Shaan</h1>

If name = "Rahul":
<h1>Hello Rahul</h1>

So EJS is basically:

HTML + JavaScript inside HTML.

 ------------------------------------- let's dig deeper -----------------------------------------------




Step 1: Everything in JavaScript is an Object ---

When you write:

const user = {
    name: "Shaan"
};

You think you created this:

user
---------
name : "Shaan"

But JavaScript actually creates something like

user
---------
name : "Shaan"

      |
      |
      V
Object Prototype

That arrow is hidden.




Step 2: What is Object Prototype?

Think of Object Prototype as the father of every normal object.

For example:
const person = {};
const car = {};
const phone = {};

All three secretly point to the same object.

person ----\
             \
car ---------> Object Prototype
             /
phone -------/

There is only one Object Prototype.
Every ordinary object shares it.




Step 3: What's inside Object Prototype?

const person = {};
console.log(person);

Looks empty.
{}

But try
console.log(person.toString());
It works.

Try
console.log(person.hasOwnProperty("name"));
Works.

Try
console.log(person.valueOf());
Works.

Where are these methods?
Not inside person.
They're inside Object Prototype.


Think of it like this:
Object Prototype
--------------------
toString()
valueOf()
hasOwnProperty()
isPrototypeOf()
propertyIsEnumerable()
...
Every object gets access to these methods.



Step 4: Why?

Imagine JavaScript did this.

person

name
toString()
valueOf()
hasOwnProperty()

car

brand
toString()
valueOf()
hasOwnProperty()

phone

model
toString()
valueOf()
hasOwnProperty()

Thousands of duplicate methods.

Instead JavaScript says
Let's keep only one copy.

So

person

↓

Object Prototype

toString()
valueOf()
hasOwnProperty()




Step 5: How JavaScript searches

Suppose:
const person = {
    name: "Shaan"
};

console.log(person.hasOwnProperty("name"));

JavaScript searches like this.

First - 

person
name

Does hasOwnProperty exist?
No.

So JavaScript follows the arrow.

Object Prototype
hasOwnProperty()

Found.
Execute it.



Step 6: Another Example:

const person = {
    name: "Shaan"
};
console.log(person.toString());

Search order

person

↓

Object Prototype

↓

Found toString()



Step 7: But where did Object Prototype come from?

Now comes the interesting part.
Everything in JavaScript is created using constructors.

When you write
const person = {};

JavaScript internally behaves roughly like this:
const person = new Object();

So you're actually creating an instance of Object.
And every instance of Object automatically gets linked to
Object.prototype

Notice the difference:

Object          ← constructor function

Object.prototype    ← an object

People confuse these two all the time.



Step 8: What is Object.prototype?

It is literally an object.

console.log(Object.prototype);

You'll see something like

{
    constructor: Object,
    hasOwnProperty(),
    toString(),
    valueOf(),
    ...
}

That's the object every normal object inherits from.





Step 9: Visual Diagram
Object Constructor
        │
        │
        ▼
Object.prototype
--------------------------------
toString()
valueOf()
hasOwnProperty()
isPrototypeOf()
--------------------------------

        ▲
        │
        │
person
{name:"Shaan"}

{brand:"BMW"}

phone
{model:"iPhone"}

All arrows point to the same prototype object.







Step 10: Let's prove it
const person = {};
const car = {};

console.log(person.toString === car.toString);

Output

true

Why?

Because both are using the exact same function from Object.prototype.

There aren't two copies.

Step 11: One thing that surprises everyone

You can actually add your own method to Object.prototype.


Object.prototype.sayHello = function () {
    console.log("Hello");
};

const person = {};
const car = {};
person.sayHello();
car.sayHello();


Output:

Hello
Hello

Why?

Because both objects inherit from Object.prototype.
In practice, you should avoid modifying Object.prototype in real projects, because it affects every object in your program and can cause unexpected behavior. It's useful here only to demonstrate how inheritance through prototypes works.


The big picture

When you create a normal object:

const user = {
    name: "Shaan"
};

JavaScript secretly creates a hidden link like this:

user
{name:"Shaan"}

        │
        ▼

Object.prototype
--------------------------
toString()
valueOf()
hasOwnProperty()
...

When you ask for a property or method:

JavaScript checks user.
If it isn't there, it checks Object.prototype.
If it's still not found, it continues up the prototype chain until it reaches null.
