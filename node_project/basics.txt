So these are the two options:

CommonJS (what we're currently using):
package.json ---
{
  "name": "node_project"
}

--- server.js --- 

const express = require("express");
✅ No "type": "module" needed.


--- ES Modules ---

package.json

{
  "name": "node_project",
  "type": "module"
}

--- server.js --- 

import express from "express";
✅ Works.














Think of server.js as the main entry point of your backend.

It's similar to how:

index.html is the starting point of a website.
main() is the starting point of a C/C++ program.
main.py is often the starting point of a Python project.

Everything eventually connects back to the file that starts Node:
node server.js
















----------- .ejs FILE extension ----------------

So what is .ejs?
EJS = Embedded JavaScript
It is a template engine for Express.

A normal HTML file is static:
<h1>Hello User</h1>

But suppose you want:
User A → "Hello Shaan"
User B → "Hello Rahul"

You would need to generate HTML dynamically.
That's where EJS comes in.

Example:
<h1>Hello <%= name %></h1>

Express:
res.render("home.ejs", {
    name: "Shaan"
});

Output:
<h1>Hello Shaan</h1>

If name = "Rahul":
<h1>Hello Rahul</h1>

So EJS is basically:

HTML + JavaScript inside HTML.
