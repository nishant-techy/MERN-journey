Step 7: But where did Object Prototype come from?

Now comes the interesting part.
Everything in JavaScript is created using constructors.

When you write
const person = {};

JavaScript internally behaves roughly like this:
const person = new Object();

So you're actually creating an instance of Object.
And every instance of Object automatically gets linked to
Object.prototype

Notice the difference:

Object          ← constructor function

Object.prototype    ← an object

People confuse these two all the time.



Step 8: What is Object.prototype?

    hasOwnProperty(),
    toString(),
    valueOf(),
    ...
}

That's the object every normal object inherits from.

Step 10: Let's prove it

Step 11: One thing that surprises everyone

You can actually add your own method to Object.prototype.


Object.prototype.sayHello = function () {
    console.log("Hello");
};

const person = {};
const car = {};
person.sayHello();
car.sayHello();


Output:

Hello
Hello

Why?

Because both objects inherit from Object.prototype.
In practice, you should avoid modifying Object.prototype in real projects, because it affects every object in your program and can cause unexpected behavior. It's useful here only to demonstrate how inheritance through prototypes works.

------ INHERITANCE -------

Inheritance is the mechanism that allows us to create new classes on the basis of already existing classes. 


Inheritance allows one class (called the child or derived class) to acquire the properties and methods of another class (called the parent or base class).

Think of it like this:

Parent: Vehicle
Children: Car, Bike, Truck

All vehicles have common features like start() and stop(), so instead of writing those methods repeatedly, the child classes inherit them.



for example (if we create the class of student and teacher):

class Person {
	constructor(name, age) {
		console.log("person class constructor");
		this.name = name;
		this.age = age;
	}
	talk() {
		console.log(`Hi, I am ${this.name}`);
	}
}


class Student extends Person {
	constructor(name, age, marks) {
		console.log("students class constructor");
		super(name, age); // parent class constructor is being called
		this.marks = marks;
	}
}


class Teacher extends Person {
	constructor(name, age, subject) {
		super(name, age) // parent class constructor is being called
		this.subject = subject;

	}
}



















---------------- BASIC Q&A ----------------

1). What is Object-Oriented Programming (OOP)?

Object-Oriented Programming (OOP) is a way of writing code where we create objects from classes.

Think of a class as a blueprint, and an object as the real thing made from that blueprint.

::Real-life Example::

Imagine you're building houses.
* Blueprint → Class
* Actual House → Object

One blueprint can be used to build many houses.


Similarly,

class Car {
    constructor(name, color) {
        this.name = name;
        this.color = color;
    }
}

const car1 = new Car("BMW", "Black");
const car2 = new Car("Audi", "White");

Here,
* Car → Class (Blueprint)
* car1 and car2 → Objects


Interview Answer:
OOP is a programming style where we create classes and objects to organize code, making it reusable and easier to maintain.
















2. Benefits of OOP

* Improved code organization
* Reusability
* Maintainability
* Closeness to real-world objects

Let's understand each:

4. What is a Constructor Function?

A constructor function creates objects.

Before ES6 classes existed, JavaScript used constructor functions.

Example: 
function Student(name, age){
    this.name = name;
    this.age = age;
}

const s1 = new Student("Rahul",22);

Output -- 
s1.name
Rahul

Here
new Student(...)
creates the object.


Same thing using Class:
class Student{
   constructor(name, age){
      this.name = name;
      this.age = age;
   }
}

Both create objects.
Classes are simply a cleaner syntax.
